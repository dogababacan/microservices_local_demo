Modularity Demo: Analytics Service

Analytics Service demonstrates modularity. It was added without changing the services that publish the events.

It is a side observer, not part of the main checkout chain. Checkout still moves from Order to Inventory to Payment to Notification even if Analytics Service is stopped.

  • Service inbox: analytics_service_events_queue
  • Subscribes to note labels: order.created, inventory.reserved, inventory.failed, payment.completed, payment.failed
  • Publishes events: none
  • Teaching point: existing services do not know Analytics Service exists.

Advanced note: different queues bound to the same routing key each receive their own copy of the event. This is why Inventory and Analytics can both receive order.created.

Trace One Checkout Through Services

  1. API Gateway receives HTTP request
  2. Order Service creates order
  3. Order Service publishes order.created
  4. Inventory Service consumes order.created
  5. Analytics Service also observes order.created from its own queue
  6. Inventory Service publishes inventory.reserved
  7. Payment Service consumes inventory.reserved
  8. Analytics Service also observes inventory.reserved from its own queue
  9. Payment Service publishes payment.completed
  10. Notification Service consumes payment.completed
  11. Analytics Service also observes payment.completed from its own queue

What each service does NOT know

  • Order Service does not know Inventory Service exists.
  • Inventory Service does not know Payment Service exists.
  • Payment Service does not know Notification Service exists.
  • No existing service knows Analytics Service exists.
  • Services only know event names and RabbitMQ connection details.

Code Map

  • src/index.js: starts the HTTP server or service process and contains the main service behavior
  • src/rabbitmq.js: connects to RabbitMQ and contains publish/consume helper functions
  • Dockerfile: tells Docker how to run this service
  • package.json: lists the Node.js dependencies and start script
  • docker-compose.yml: starts all services and RabbitMQ together
